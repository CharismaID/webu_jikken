## 📌 Описание задания
В данной лабораторной работе реализовано подключение веб-приложения к реляционной базе данных MySQL:
* Настроена инфраструктура Docker: добавлен контейнер с базой данных MySQL и панелью управления Adminer. Написан собственный `Dockerfile` для установки расширений `pdo` и `pdo_mysql` в PHP.
* Реализовано безопасное подключение к БД с использованием `PDO`.
* Создан ООП-класс `Student` для автоматического создания таблицы `students` и выполнения CRUD-операций.
* Данные из формы регистрации теперь сохраняются не в текстовый файл, а в полноценную таблицу базы данных.
* **Штрафное/Дополнительное задание:** * В таблицу добавлено поле `created_at` (дата и время регистрации).
  * Реализован вывод записей на странице списка с сортировкой по дате (свежие сверху) через SQL (`ORDER BY`).
  * Реализован подсчет общего количества студентов через SQL-функцию `COUNT()`.
  * Добавлен SQL-фильтр (вывод только совершеннолетних студентов) через параметр в URL (`WHERE age >= 18`).

Результат доступен по адресу: [http://localhost:8080](http://localhost:8080).

📂 Содержимое проекта
    • docker-compose.yml — описание сервиса Nginx, PHP, MySQL и Adminer.
    • Dockerfile — инструкции для сборки PHP-контейнера с нужными MySQL-драйверами.
    • www/db.php — скрипт подключения к базе данных через PDO.
    • default.conf — конфигурация Nginx для обработки PHP.
    • composer.json / composer.lock / www/vendor/ — файлы пакетного менеджера Composer и библиотека Guzzle.
    • www/ApiClient.php — ООП-класс для выполнения внешних HTTP-запросов к API.
    • www/UserInfo.php — ООП-класс для сбора данных о браузере пользователя.
    • www/form.html — HTML-форма регистрации студента (имя, возраст, факультет, форма обучения).
    • www/Student.php — класс для работы с таблицей студентов (создание, добавление, выборка, фильтрация, подсчет).
    • www/process.php — скрипт обработки формы, валидации, работы с сессиями и сохранения в БД.
    • www/index.php — главная страница: вывод данных последней регистрации, информации о пользователе и данных из API.
    • www/view.php — страница просмотра: чтение и вывод всех студентов из базы данных с фильтрацией.
    • www/api_update.php — скрипт-обработчик для асинхронного JS fetch-запроса.
    • www/api_cache.json — файл для кеширования результатов от внешнего API.
