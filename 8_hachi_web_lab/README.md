## 📌 Описание задания
В данной лабораторной работе реализована система автоматического тестирования для проверки корректности работы бизнес-логики и интеграций на примере **системы регистрации студентов**:

* **Unit-тестирование:** Написаны тесты для класса `Student`, проверяющие логику валидации и сохранения данных студентов.
* **Mock-объекты:** Использование заглушек для `PDO` и `PDOStatement`, что позволяет тестировать класс в изоляции от реальной базы данных.
* **Тестирование API:** Проверка HTTP-взаимодействий через `Guzzle`. Использование `MockHandler` для имитации ответов от внешних сервисов без выполнения реальных сетевых запросов.
* **Окружение:** Настройка изоляции тестовой среды через файлы конфигурации и Docker-volumes.

### 💀 Штрафное задание (Реализовано):
1. **Интеграционное тестирование:** Написан тест, который подключается к реальному контейнеру MySQL (`db`) для проверки корректности SQL-запросов и структуры таблицы.
2. **Тестовая конфигурация:** Использование файла `.env.test` для переключения на тестовую базу данных внутри Docker-сети.
3. **Проверка состояния:** Реализованы ассерты на проверку количества записей в таблице (`getTotalCount`) и корректности извлеченных данных (`getAll`).

## ⚙️ Как запустить проект

1. **Клонировать репозиторий и перейти в ветку:**
   ```bash
   git clone https://github.com/CharismaID/webu_jikken.git
   cd webu_jikken
   git checkout lab8

```

2. **Запустить контейнеры:**
```bash
docker-compose up -d --build

```


3. **Установить зависимости и настроить автозагрузку:**
```bash
docker exec -it lab7_php composer install
docker exec -it lab7_php composer dump-autoload

```


4. **Запуск тестов:**
```bash
# Запуск всех тестов сразу
docker exec -it lab7_php vendor/bin/phpunit tests

```
