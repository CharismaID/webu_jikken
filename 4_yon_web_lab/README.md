## Описание задания

В данной лабораторной работе реализована интеграция с внешними сервисами и использование пакетного менеджера:

- Подключение внешней библиотеки (Guzzle) с помощью Composer.
    
- Использование объектно-ориентированного подхода в PHP (создание классов ApiClient и UserInfo).
    
- Отправка HTTP-запросов к публичному API HeadHunter (раздел areas) для получения списка регионов и стран.
    
- Работа с куками (setcookie) для отслеживания времени последней активности пользователя.
    
- Сбор технической информации о пользователе (IP-адрес, User-Agent, серверное время).
    
- **Штрафное задание:** Реализовано кеширование ответов от API в локальный файл (api_cache.json с проверкой актуальности 5 минут) и асинхронное обновление данных на странице с помощью JavaScript Fetch API без перезагрузки.

## Как запустить проект

1. Клонировать репозиторий:
    
    Bash
    
    ```
    git clone https://github.com/CharismaID/webu_jikken.git
    cd web_labs
    ```
    
2. Запустить контейнеры (автоматически применит настройки Dockerfile и docker-compose):
    
    Bash
    
    ```
    docker-compose up -d --build
    ```
    
3. Открыть проект в браузере:
    
    `http://localhost:8080` (или `http://localhost:8000` в зависимости от настроек вашего порта)

## Содержимое проекта

- **docker-compose.yml** — описание сервисов (Nginx, PHP-FPM).
    
- **default.conf** — конфигурация Nginx для обработки PHP-запросов.
    
- **composer.json / composer.lock** — файлы управления зависимостями и описание библиотеки Guzzle.
    
- **ApiClient.php** — класс-обертка для выполнения HTTP-запросов к внешним API.
    
- **UserInfo.php** — класс для получения информации о текущем сеансе пользователя.
    
- **form.html** — форма регистрации абитуриента.
    
- **process.php** — серверная обработка данных формы, валидация, работа с сессиями и сохранение в файл.
    
- **index.php** — главная страница: отображение данных анкеты, информации о системе и списка регионов из API.
    
- **view.php** — страница просмотра истории всех зарегистрированных заявок из текстового файла.
    
- **api_update.php** — скрипт для обработки фоновых запросов (Fetch API) и обновления кеша.
    
- **api_cache.json** — файл временного хранения данных API для снижения нагрузки на внешнюю сеть.
