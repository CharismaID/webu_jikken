## 📌 Описание задания

В данной лабораторной работе реализована система непрерывной интеграции (CI) для автоматизации проверки корректности работы приложения при каждом обновлении кода:

* **CI/CD Pipeline:** Настроен рабочий процесс в GitHub Actions, который автоматически собирает Docker-контейнеры и запускает тесты при каждом Push-событии в ветку `main`.
* **Автоматизированное тестирование:** * **Unit-тесты:** Проверка логики класса `Student` с использованием Mock-объектов для изоляции от БД.
* **Интеграционные тесты:** Полный цикл проверки записи данных в реальную БД MySQL внутри Docker-окружения CI.


* 
**Окружение:** Использование конфигурационного файла `.env.test` для настройки доступа к тестовой базе данных `test_db` в облачной среде.



### 💀 Штрафное задание (Реализовано):

1. **Разделение пайплайна:** Процесс разделен на этапы сборки (`build`) и тестирования (`test`).
2. **Ожидание сервисов:** Добавлена задержка (`sleep`) для корректной инициализации БД перед началом тестов.
3. **Логирование:** Настроен автоматический вывод логов Docker-контейнеров в случае падения тестов для быстрой диагностики.
4. **Мониторинг:** В пайплайн включена проверка состояния запущенных контейнеров (`docker ps`).

## ⚙️ Как запустить проект локально

1. **Клонировать репозиторий:**
```bash
git clone https://github.com/CharismaID/webu_jikken.git
cd webu_jikken

```


2. **Запустить контейнеры:**
```bash
docker-compose up -d --build

```


3. **Настройка зависимостей:**
```bash
# Если папка vendor не включена в репозиторий:
docker exec -it lab7_php composer install
# Обновление автозагрузки:
docker exec -it lab7_php composer dump-autoload

```


4. **Запуск тестов вручную:**
```bash
docker exec -it lab7_php vendor/bin/phpunit tests

```

## 📂 Содержимое проекта

* `.github/workflows/ci.yml` — конфигурация автоматизации GitHub Actions.
* `www/Student.php` — основной класс бизнес-логики (регистрация студентов).
* `tests/StudentTest.php` — набор Unit и интеграционных тестов.
* 
`.env.test` — параметры подключения к тестовой БД.


* `docker-compose.yml` — описание инфраструктуры (PHP, MySQL, Redis и др.).
