## 📌 Описание задания
В данной лабораторной работе реализована отказоустойчивая асинхронная архитектура для регистрации студентов. Система разделена на независимые компоненты, которые взаимодействуют через брокеры сообщений, что позволяет обрабатывать входящие анкеты без задержек для пользователя.

* **Producer (Отправитель):** Скрипт `send.php` принимает данные анкеты из веб-формы и мгновенно отправляет их в топик **Apache Kafka**. Это гарантирует быструю отдачу HTTP-ответа пользователю.
* **Consumer (Обработчик/Worker):** Фоновый процесс `worker.php` асинхронно извлекает данные из Kafka, имитирует проверку и сохраняет их в реляционную базу данных **MySQL**.
* **Инфраструктура:** Полностью контейнеризированная среда (Docker), включающая Zookeeper, Kafka, RabbitMQ, MySQL и связку Nginx + PHP-FPM.

### Штрафное задание (Реализовано):
1.  **Dead Letter Queue (DLQ) через RabbitMQ:** Система использует два разных брокера. Если во время обработки анкеты в воркере происходит критический сбой (в коде имитируется вероятность ошибки 30%), данные не теряются. Они автоматически перенаправляются в очередь ошибок **RabbitMQ** (`lab7_errors`) для ручного разбора или повторной обработки.
2.  **Dashboard мониторинга:** На главной странице (`index.php`) отображается живая статистика:
    * Количество анкет, ожидающих обработки в Kafka.
    * Количество «упавших» заявок в RabbitMQ (данные подтягиваются напрямую через RabbitMQ Management API).

## ⚙️ Как запустить проект

1.  **Клонировать репозиторий:**
    ```bash
    git clone https://github.com/CharismaID/webu_jikken.git
    cd webu_jikken
    ```

2.  **Запустить инфраструктуру:**
    ```bash
    docker-compose up -d --build
    ```

3.  **Установить зависимости (PHP-библиотеки):**
    ```bash
    docker exec -it lab7_php composer install
    ```

4.  **Запустить воркер (обработчик):**
    В отдельном терминале запустите процесс, который будет слушать очереди:
    ```bash
    docker exec -it lab7_php php worker.php
    ```

5.  **Открыть приложение:**
    [http://localhost:8080](http://localhost:8080)

## 📂 Структура проекта

* `docker-compose.yml` — описание всех сервисов (Kafka, Rabbit, MySQL, PHP).
* `www/QueueManager.php` — класс-обертка для работы с Kafka (основной поток) и RabbitMQ (очередь ошибок).
* `www/Student.php` — модель для работы с таблицей студентов в MySQL.
* `www/send.php` — точка входа для данных (Kafka Producer).
* `www/worker.php` — скрипт-потребитель (Kafka Consumer + RabbitMQ Error Producer).
* `www/index.php` — мониторинг состояния очередей.
* `www/form.html` — форма регистрации студента.
